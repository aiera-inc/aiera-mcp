# Aiera MCP Tools

MCP server and reusable Python package exposing Aiera financial data API endpoints as MCP tools.

## Quick Reference

- **Language**: Python 3.11+
- **Package manager**: [uv](https://docs.astral.sh/uv/)
- **Entrypoint (standalone)**: `uv run entrypoint.py` or `aiera-mcp` (installed script)
- **Transport**: stdio (default), also supports sse and streamable-http via `MCP_TRANSPORT` env var
- **Required env var**: `AIERA_API_KEY`

## Tool Count

39 registered tools across 8 categories: events, filings, equities, company_docs, third_bridge, research, search, web.

## Architecture Patterns

- **Tool definition**: Each domain module has `models.py` (Pydantic Args + Response models) and `tools.py` (async tool functions). Tool descriptions come from the Args model docstrings, auto-populated in `registry.py`.
- **Registry**: `TOOL_REGISTRY` in `registry.py` is the single source of truth — maps tool names to `{display_name, description, input_schema, function, args_model, category, read_only, destructive}`.
- **Auth**: Dual-mode — environment variable (`AIERA_API_KEY`) or pluggable OAuth provider via `set_api_key_provider()`. Provider takes priority; env var is fallback.
- **Context injection**: Downstream consumers (e.g. aiera-public-mcp) can inject custom request headers and error handlers via `set_request_context_provider()` and `set_error_handler()` without modifying tool code.
- **HTTP clients**: Shared `httpx.AsyncClient` with connection pooling. Lambda-compatible singleton via `get_lambda_http_client()`.
- **All tools are read-only and non-destructive.**

## Configuration

All settings via env vars or `.env` file (see `.env.example`):

| Variable                | Default                         | Description                                                 |
|-------------------------|---------------------------------|-------------------------------------------------------------|
| `AIERA_API_KEY`         | (required)                      | API key                                                     |
| `AIERA_BASE_URL`        | `https://graphql.aiera.com/api` | API base URL                                                |
| `DEFAULT_PAGE_SIZE`     | `25`                            | Default pagination size (aiera-api clamps to 25 regardless) |
| `DEFAULT_MAX_PAGE_SIZE` | `25`                            | Max pagination size (aiera-api enforces this server-side)   |
| `HTTP_TIMEOUT`          | `30.0`                          | Request timeout (seconds)                                   |
| `LOG_LEVEL`             | `INFO`                          | Logging level                                               |

## Adding a New Tool

1. Create or extend a domain module under `aiera_mcp/tools/<domain>/`
2. Define Args model (Pydantic, with docstring — this becomes the tool description) and Response model in `models.py`
3. Implement async tool function in `tools.py`
4. Export from domain `__init__.py`
5. Register in `aiera_mcp/tools/registry.py` — add import and `TOOL_REGISTRY` entry
6. Add to `AVAILABLE_TOOLS` list and relevant tool group in `aiera_mcp/__init__.py`
7. Add to `__all__` exports in `aiera_mcp/__init__.py`

## Key Conventions

- Bloomberg tickers use `TICKER:COUNTRY` format (e.g. `AAPL:US`). The `correct_bloomberg_ticker()` utility auto-corrects common mistakes.
- Comma-separated lists with no spaces for multi-value parameters.
- Dates in ISO format (`YYYY-MM-DD`).
- Tool functions receive a parsed Pydantic args model and return a response model or dict.
- Search tools use OpenSearch with embedding and hybrid pipelines (constants `EMBEDDING_SEARCH_PIPELINE`, `HYBRID_SEARCH_PIPELINE` in `__init__.py`).
